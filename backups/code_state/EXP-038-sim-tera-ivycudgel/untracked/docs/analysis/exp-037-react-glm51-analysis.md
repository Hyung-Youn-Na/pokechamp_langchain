## 2. 핵심 발견

### 2.1 결정적 차이: protect 시뮬레이션 감소 + OHKO 언급 증가

fix2 적용 후 protect 관련 신호가 뚜렷이 변했다:

| 신호 (EXP-036 → 037) | 값 | 해석 |
|------|-----|------|
| `tool_result` protect 언급 | 71 → **15** | protect 시나리오 시뮬레이션 급감 |
| LLM `protect` 언급 턴 | 161 → **100** | protect 논의 감소 |
| LLM `ohko/guaranteed` 언급 | 323 → **371** | OHKO 보고 증가 |
| LLM `life orb` 언급 | 3 → 3 | metamon 팀에 LifeOrb 거의 없음 → LifeOrb 효과 미미 |

**fix2의 실질 작동은 protect 쪽**. LifeOrb는 metamon 팀 구성상 빈도가 낮아 거의 무효.

### 2.2 가설 검증: 보수화? — 지표상 입증 안 됨

protect를 올바르게 0데미지로 모델링하자 agent가 보수화(setup/회복/스위치 남용)했다는 가설을 검증:

| 보수화 지표 (EXP-036 → 037) | 값 |
|------|-----|
| setup 무브 언급 턴 (Swords Dance/Calm Mind/...) | 4.9% → **4.4%** (감소) |
| 회복 무브 언급 턴 (Roost/Recover/...) | 8.4% → **8.4%** (동일) |
| switch 언급 | 389 → **413** (미증) |

**setup/heal 비율 변화 없음** → 보수화 가설 약함. ohko 언급 증가만 뚜렷하나, 이것이 protect fix와 어떻게 연결되는지 명확한 코드 경로 부재.

### 2.3 인과 불명 + 비유의 → 표본 분산 가능성

- 코드 결함: 없음(단위 테스트 3/3 PASS, protect 0데미지는 showdown 정확 스펙).
- 보수화: 지표상 입증 안 됨.
- **−13.3pp는 z≈−1.17, p≈0.24로 비유의**. EXP-036과 비동일 매칭(다른 팀 시드)이라 paired 비교 불가 — 표본 분산(팀 매칭 운)일 가능성 존재.

## 부록 — fix2 기각 논리 불충분

EXP-035는 **코드 결함**(동적 위력 중복 보정)으로 기각됐다. 반면 fix2는:
- 코드 정확(protect 0데미지 = showdown 스펙, LifeOrb 1.3× = 정확).
- 단위 테스트 3/3 PASS.
- 승률 하락은 비유의(z≈−1.17) + 인과 불명.

따라서 "해롭다" 단정 근거 부족. 단, "이득 증거도 없음". 올바른 동작이므로 **유지하되 EXP-038 누적 평가로 순 효과 확정**하는 것이 시뮬레이터 무결성 관점에서 합리적.
