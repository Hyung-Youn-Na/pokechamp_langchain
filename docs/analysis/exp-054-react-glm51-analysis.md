## 3. 역행 원인 분석

1. **smoke에서 정확 동작은 확인(EXP-054 변경 자체는 버그 수정)**: opp 중도 HP 시뮬 5시나리오 — `0.27→1%`(27−26), `0.15→0%` KO, `0.15→15%`(pmed=0). 기존 `100-pmed`는 모두 **거짓 생존**(74%/85%/100%)이었음. 변경은 의도대로 동작.
2. **"정확성=승률"의 반례**: EXP-050a에선 oracle 절대 데미지 정확화가 +16pp였으나, simulate_turn 잔여 HP 정확화는 **−(20~13)pp 역행**. 정확한 수치 자체가 승률↑로 직결되지 않음.
3. **가설 (3가지, 미확정)**:
   - (a) **낙관적 거짓 생존이 공격적 KO 유도에 유리** — 050a/050e의 `100-pmed`(opp가 항상 풀피로 보고)가 "아직 KO 안 났다, 계속 두드려라"는 낙관 신호로 작용, 빠른 KO 압박 → 역설적 우위(EXP-052 분석의 "과소 집계 거짓 우위"와 동일 패턴).
   - (b) **정확한 잔여 HP가 매턴 변동 신호로 과부하** — opp/player HP가 턴마다 변하며 strategy_synthesis가 과반응, my_plan/결정 교란.
   - (c) **N=10 노이즈** — z<1.5로 유의성 부족.
4. **시사**: 도구 정확성(수치)만 단독 적용 시 역행. 정확한 수치를 LLM이 제대로 활용하려면 **결정 품질 구조**(strategy_synthesis 교정 프레임, EXP-055)와 결합 필요 — 수치가 "사실"로 과신되지 않도록 프레이밍.

smoke에서 simulate_turn 결과가 opp/player 현재 HP에서 정확 차감. json_parse_failures 1(게이트 임계 이내). 도구 자체 회귀 없음.

## 4. 결론 / 후속

- **기각**: 8/10·net +1 게이트 미달 + 050a 대비 −3 역행.
- **HP 맥락 보존 자체는 정확성↑(거짓 생존 제거 검증)** 이나, 단독 적용 시 승률↓. "정보 정확화 ≠ 항상 승률↑" 재확증(메모리 `react-memory-lever-ceiling`과 일관).
- **EXP-054 코드는 회수**(050e 순수 베이스에서 EXP-055 시작) — 단, **"HP 보존 + 교정 프레임 결합" 시너지**는 미래 후보로 기록(EXP-055 통과 시 그 위에서 HP 보존 재평가 가능).
- 후속(EXP-055): `react_agent.py` strategy_synthesis 교정 프레임(헤더 `## Tool Results:`→ESTIMATES + STRATEGY_SYSTEM_PROMPT 도구 신뢰도 지시) — 결정 품질 레버, 050e 순수 베이스.
