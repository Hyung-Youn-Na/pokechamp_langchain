## 0. TL;DR

**fix3(tera/ivycudgel)는 승률 향상도 거짓 OHKO 감소도 이끌지 못함.** 승률 66.7% (20/30) = EXP-037과 동일. **z(037 vs 038)=0.00**으로 fix3 단독 효과 이론상 0이 실제로 확인됐다.

근거는 명확: **tera 언급 0.7% (16/2254 턴), ivycudgel/ogerpon 등장 3배틀** — fix3가 다루는 기믹(Terastallize·Ogerpon)이 이 매칭(abyssal 상대)에서 극히 드물어 직접 영향 범위가 거의 0이다.

→ **시리즈 종합**: `fix1`(80%) > `fix1+fix2`(66.7%) = `fix1+fix2+fix3`(66.7%). **fix2(protect)가 시리즈 하락의 원인**, fix3는 회복시키지 못함. fix1 단독이 최고 성능.

## 2. 핵심 발견

### 2.1 fix3는 이 매칭에서 사실상 무효

fix3가 다루는 기믹의 발동 빈도:
- **Terastallize**: LLM tera 언급 **0.7%** (16/2254 턴). abyssal은 휴리스틱이라 tera 미사용, agent도 tera 거의 안 함.
- **Ogerpon/ivycudgel**: 도구 입력 12/8회, **등장 배틀 3개** (862/867/871). 30배틀 중 10%.

→ fix3(tera 상성·STAB tera·ivycudgel 타입·위력)는 **발동 조건 자체가 거의 안 성립**하여 승률·거짓 OHKO에 measurable한 영향을 줄 수 없었다. 단위 테스트(4/4 PASS)로 코드 정확성은 확인됐으나, 이 매칭에서의 효과 측정은 불가.

### 2.2 시리즈 하락은 fix2가 원인 (fix3가 회복 못 함)

| 비교 | 승률 | z | 해석 |
|------|------|---|------|
| EXP-036(f1) vs EXP-037(f1+2) | 80%→66.7% | −1.17 | fix2 하락 (비유의) |
| EXP-037(f1+2) vs EXP-038(f1+2+3) | 66.7%→66.7% | **0.00** | **fix3 효과 0** |
| EXP-036(f1) vs EXP-038(f1+2+3) | 80%→66.7% | −1.17 | 시리즈 순 하락 |

EXP-037과 EXP-038이 동일(66.7%)이므로, fix3(무효) 추가가 fix2의 −13.3pp를 회복시키지 못했다. 즉 **fix2가 시리즈 하락의 원인**으로 EXP-037 분석의 "보류"를 넘어 **하락 일관성**이 강화됐다. 단, 두 측정 모두 비동일 매칭(다른 팀 시드)이라 paired 비교는 아님.

### 2.3 근본 통찰: 시뮬레이터 정확도 ≠ 승률

- **fix1(priority/protosynthesis)**: 행동 순서 → 매 턴 영향 → 승률 +3.3pp. ✅ 효과.
- **fix2(protect/item)**: protect 정확화 → agent 행동 변화(보수화 의심) → −13.3pp. 🔴 역효과.
- **fix3(tera/ivycudgel)**: 드문 기믹 → 효과 측정 불가. ⚪ 무효.

시뮬레이터를 "정확하게" 고치는 것만으로 승률이 오르지 않는다. 정확해진 sim을 agent가 **어떻게 소비하는가**(과신 vs 비판적 검증)가 승률을 좌우한다. 이는 EXP-034 P0-1 "시뮬레이터 오답 맹신" 문제의 연장 — fix는 sim 정확도를 높였지만 agent의 맹신 패턴은 그대로.

## 3. 문제점 분석

### 🔴 P0-1: fix2(protect) 역효과 미해결

| 항목 | EXP-038 | EXP-037 | EXP-036 |
|------|---------|---------|---------|
| 승률 | 66.7% | 66.7% | 80.0% |
| mid 구간 | 68.4% | 77.8% | 85.0% |

**근본 원인**: EXP-037에서 보류했던 fix2 하락이 EXP-038(037과 동일 66.7%)에서 일관성 확인. 비유의(z≈−1.17)지만 두 독립 측정이 동일 하락 → 표본 분산보다 실제 하락 가능성 상승. 올바른 동작(protect 0데미지)이 agent의 protect 예측 과대·보수화와 결합해 역효과.
**개선 방안**: fix2 기각 후 fix1 단독(80%)으로 회귀, 또는 동일 팀 시드 paired 재실험으로 인과 확정.

### 🟡 P1-1: 시뮬레이터 정확도 → 승률 전이 실패

**근본 원인**: fix1/2/3 모두 sim을 정확하게 만들었으나, fix1만 승률로 전이. agent가 calc/simulate 결과를 비판 없이 수용(EXP-034 P0-1)하는 패턴이 sim 정확도 향상의 이득을 상쇄.
**개선 방안**: sim 정확도와 직교하는 **agent 측 개선** — 도구 결과에 대한 self-verification, 팀/matchup 분석 활성화(EXP-034 P1-1, 도구 비중 <3%).

### 🟢 P2-1: fix3 효과 측정 불가 (매칭 한계)

tera 0.7%·Ogerpon 3배틀로 fix3 영향 범위 0. 올바른 코드(단위 테스트 PASS)이나 tera 빈번한 매칭(LLM 상대 등)에서 재검증 필요.

## 부록 — 시리즈 교훈

**가설 "시뮬레이터 코어 버그 fix → 승률 90% 도달"은 부분적만 성립.**
- 매 턴 영향(fix1 priority): 전이 ✅.
- 조건부 기믹(fix2 protect, fix3 tera): 비전이 또는 역효과.
- 근본 병목은 sim 정확도가 아니라 **agent의 도구 결과 소비 방식**(맹신). 다음 레버는 sim이 아닌 agent 측(self-verification, 팀 분석).
